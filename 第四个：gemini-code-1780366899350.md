# Task: 生成核心 AI 审核工作台页面（左右分栏、状态联动布局）

## 1. 页面基本结构
整体采用 1:1 或 2:3 的**左右双栏分屏布局**，高度铺满屏幕：
- **左侧栏：PDF/图片预览区**：
  - 模拟一个 PDF 阅读器容器（有放大、缩小、旋转等占位工具栏）。
  - 中间显示一份金属材质报告单的图片/PDF 预览占位。
- **右侧栏：AI 识别结果与审核决策面板**：
  - 顶部带一个明显的“状态标签区域”，用来模拟和切换系统输出的两种核心结果状态：【PASS】和【FAIL】。

## 2. 状态联动逻辑与 UI 表现（核心）
请设计可以动态切换的两个状态视图：

### 状态 A：当 AI 结果为 【PASS】 时
- **状态卡片**：顶部高亮显示绿色大横幅：`AI Result: PASS`。
- **业务提示**：文案提示“系统已自动通过，不依赖人工审批，可进入后续业务流程”，字体颜色为绿色。
- **提取字段展示**：使用结构化 Key-Value 列表展示 AI 提取的数据（只读状态）：
  - 材质牌号 (Material Grade): ASTM A182 F316L
  - 炉号 (Heat Number): A123456
  - 化学成分判定: OK (绿色)
  - 力学性能判定: OK (绿色)
- **下方操作区**：**无需任何人工操作按钮**。

### 状态 B：当 AI 结果为 【FAIL】 时
- **状态卡片**：顶部高亮显示红色大横幅：`AI Result: FAIL`。
- **原因警示框**：使用红色 Alert 组件，列出具体失败原因（列表形式）：
  - ❌ 炉号 (Heat Number) 缺失或无法可靠识别
  - ❌ S元素（硫）化学成分超出预设标准范围
- **提取数据详情**：可编辑或带高亮标红的表格，展示有争议的指标。
- **人工审核操作动作条（底部固定）**：
  - 按钮 1（红色主按钮）：`确认不合格 (Confirm Fail)`
  - 按钮 2（黄色边框按钮）：`强转为合格 (Override to Pass)`
  - 按钮 3（文本/修饰按钮）：`添加备注 (Add Comment)`
