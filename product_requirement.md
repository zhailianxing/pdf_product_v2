# **材质报告 AI 自动审核系统产品说明**

1. ## **项目目标**

开发一套本地部署的软件，用于自动审核材质报告（Material Certificate / EN10204 3.1等），替代现有传统OCR识别系统。

实现：

**AI自动识别 + 自动判断 + 自动输出审核结果**

核心价值：

减少人工审核工作量，提高审核效率和一致性。

1. # **产品核心逻辑**

用户上传材质报告PDF（可能有图片格式）后，系统自动完成以下流程：

1. 读取PDF内容
2. AI识别报告信息
3. 提取关键字段
4. 根据预设审核逻辑进行判断
5. 输出审核结果

系统输出两种结果：

## **PASS**

表示：

AI判断材质报告符合要求。

处理逻辑：

- 系统自动通过
- 不需要人工逐单审核
- 可进入后续业务流程

说明：

PASS是自动通过，不依赖人工审批。

## **FAIL**

表示：

AI判断报告存在问题或风险。

例如：

- Heat Number缺失
- 材质牌号不匹配
- 标准不符合要求
- 化学成分超范围
- 机械性能不达标
- 关键字段冲突
- 无法可靠识别

处理逻辑：

进入人工审核队列。

1. # **人工审核机制**

人工审核不是正常业务流程的一部分，而是质量控制机制。

## **FAIL件**

规则：

100%人工审核。

目的：

避免AI误判。

例如：

AI判断Fail，但人工确认实际没问题。

## **PASS件**

规则：

系统自动通过。

不需要逐单人工审核。

但质量部门需要定期抽检。

例如：

初期：

20%~30%

稳定后：

5%~10%

目的：

验证AI没有漏判。

即：

AI认为PASS，但实际存在问题。

核心原则：

**PASS自动通过，FAIL人工确认，PASS抽检用于质量监控。**

1. # **V1范围**

本版本包含：

✅ PDF文件上传

✅ 本地Qwen 3 VL模型调用

✅ PDF内容识别

✅ 材质报告关键字段提取

✅ AI审核判断

✅ PASS自动通过

✅ FAIL进入人工审核

✅ PASS抽检机制

✅ 审核记录保存

✅ 审核日志与统计

本版本暂不包含：

❌ 炉号打标识别系统对接

（未来可能整合）

❌ ERP / QMS / MES接口

❌ 自动模型训练

❌ 多文件自动关联

例如：

材质报告 + 炉号照片自动匹配

1. # **识别内容（第一版）**

第一版重点识别：

基础信息：

- Supplier
- Certificate Number
- Material Grade
- Standard
- Heat Number
- Batch / Lot Number
- Date

化学成分：

例如：

- C
- Si
- Mn
- P
- S
- Cr
- Ni
- Mo
- N

机械性能：

例如：

- Yield Strength
- Tensile Strength
- Elongation
- Hardness
- Impact（如有）

AI审核判断结果：

例如：

- PASS
- FAIL

以及失败原因：

- Heat Number missing
- Material mismatch
- Chemical composition out of range
- Mechanical property below requirement
- Recognition confidence too low

## **普通业务用户**

职责：

- 处理FAIL件

## **质量管理员**

职责：

- 审核FAIL件
- 抽检PASS件
- 监控模型准确率
- 调整抽检比例

1. # **前端页面要求（简版）**

一个核心审核页面。（详细需与客户再确认）

布局：

左侧：

PDF预览

右侧：

AI识别结果 + 审核结果

示例：

PASS：

```Plain
AI Result: PASS

Material Grade: ASTM A182 F316L
Heat Number: A123456
Chemical Composition: OK
Mechanical Properties: OK
```

FAIL：

```Plain
AI Result: FAIL

Reason:
- Heat Number missing
- Tensile strength below requirement
```

FAIL人工审核操作：

按钮：

- Confirm Fail
- Override to Pass
- Add Comment

PASS无需人工操作。

1. # **技术模块建议**

建议拆分为4个模块：

1. ## **文件模块**

负责：

- PDF上传
- 文件存储
- 文件管理

1. ## **AI模块**

负责：

调用本地Qwen 3 VL

输出：

- 结构化识别结果
- PASS / FAIL判断
- 原因说明

1. ## **审核模块**

负责：

- FAIL人工审核流程
- PASS抽检流程
- 审核状态管理

1. ## **数据模块**

保存：

- 原始PDF
- AI识别结果
- PASS / FAIL状态
- FAIL人工审核结果
- PASS抽检记录
- 审核日志
- 模型准确率统计

1. # **关于模型测试报告说明**

附件中的Qwen测试报告仅用于说明当前模型能力。

说明：

模型原始输出 ≠ 产品最终输出。

软件需要将模型能力封装为稳定流程：

**上传 → AI判断 → 自动结果 → 审核闭环**

1. # **当前技术原则**

当前阶段默认：

**PDF直接输入Qwen 3 VL进行识别。**

优先快速实现MVP。

后续如果扫描件效果不足，再考虑加入：

- OCR预处理
- PDF转图片预处理
- 规则增强

# **核心目的**

**用本地AI自动审核系统替代传统OCR材质报告识别系统。**
