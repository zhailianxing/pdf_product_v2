# Task: 生成系统仪表盘概览页（Dashboard）

## 1. 顶部数据卡片（KPI Cards）
横向排列 4 个轻量化统计卡片，带趋势小图标：
- 总上传报告数（如：1,248 份）
- 自动通过率 (PASS 占比，如：78.5%)
- 人工介入率 (FAIL 占比，如：21.5%)
- AI 识别平均耗时（如：1.8 秒）

## 2. 核心内容区（双栏布局）
- **左侧：待审核队列简表（Quick Queue）**：
  展示一个状态为“FAIL”的紧急待人工审核列表。字段包括：报告名称、厂商（如：博坦、轩诺）、触发报警原因（如：S元素范围异常）、上传时间、操作按钮（去审核）。
- **右侧：大模型视觉识别（VL版）评测矩阵（参考 LLM_result.md 数据）**：
  一个对比不同尺寸大模型（3-VL-8B, 3-VL-32B, 3-VL-235B 等）在各厂商报告（VDV、迪宝、轩诺、巨力、博坦）上识别准确度的状态表格。
  - 用绿色图标表示“√（完全正确）”
  - 用橙色标签表示带注释的警告（例如：“√* S元素范围有问题”、“× 化学成分识别错误”）
  - 让该表格看起来像是系统的“模型性能监控看板”。
